Network Traffic Analysis Using Wireshark

Project Overview

This project involved conducting a comprehensive forensic investigation of the "Catbomber" malware incident. Using Wireshark, I analyzed network traffic to identify compromised systems, trace malware communication patterns, and extract critical evidence of the attack.

Key Findings

• Identified two infected clients (10.5.28.229 and 10.5.28.8) through HTTP traffic filtering and TLS handshake analysis

• Extracted exfiltrated credentials including usernames, hostnames, email addresses, and passwords from captured packets

• Verified malicious files as Indicators of Compromise using VirusTotal

• Detected fake SSL certificates used in command-and-control communications

Screenshots & Analysis

Fake SSL Certificate: Post-infection TLS traffic analysis revealing fraudulent certificate with commonName "example.com" used for command-and-control communication over TCP ports 443, 447, and 449.
VirusTotal Confirmation: File hash uploaded to VirusTotal confirming 61/72 security vendors flagged cursor.png as malicious trojan (Emotet/ERPH). Verified as Indicator of Compromise.
Email Credentials Exfiltration: HTTP stream showing captured Outlook passwords and email credentials (phillip.ghent@...) being transmitted by infected client to attacker's server.
Second Infected Client (Catbomber-DC): Windows IP configuration revealing second compromised system on the network with hostname "Catbomber-DC" and IP address 10.5.28.8.
Primary Infected Client Discovery: HTTP traffic revealing hostname "Cat-Bomb-W7-PC" with IP address 10.5.28.229 (Preferred) and network configuration details of the primary compromised system.
Malware Extraction Process: Using Wireshark's Export HTTP Objects feature to extract suspicious files (cursor.png, imgpaper.png) from network traffic for forensic analysis and hash verification.

Technologies Used

Wireshark • PCAP Analysis • Network Security Monitoring • HTTP/TLS Protocol Analysis • VirusTotal • PowerShell • Forensic Analysis
