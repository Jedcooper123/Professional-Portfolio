Network Traffic Analysis Using Wireshark

Project Overview

This project involved conducting a comprehensive forensic investigation of the "Catbomber" malware incident. Using Wireshark, I analyzed network traffic to identify compromised systems, trace malware communication patterns, and extract critical evidence of the attack.

Key Findings

• Identified two infected clients (10.5.28.229 and 10.5.28.8) through HTTP traffic filtering and TLS handshake analysis.

• Extracted exfiltrated credentials including usernames, hostnames, email addresses, and passwords from captured packets.

• Verified malicious files as Indicators of Compromise using VirusTotal.

• Detected fake SSL certificates used in command-and-control communications.

Password Authentication & Hash Cracking

Project Overview

This project explores how Windows NTLMv2 authentication works on the wire and why weak passwords are dangerous. I captured NTLMv2 handshake traffic with Wireshark, exported the hashes, and used Hashcat on Kali Linux to attempt offline password cracking with a common wordlist.

Workflow

• Capture NTLMv2 logons over the network using Wireshark filters.
• Extract the server challenge and NTLMv2 response values from the authentication packets.
• Build a hashes.txt file in the correct Hashcat format and move it to Kali.
• Run Hashcat (mode 5600) with the rockyou.txt wordlist to test password strength.

Screenshots & Analysis

Capturing NTLMv2 Response
Wireshark filtering on ntlmssp to locate the NTLMv2 authentication packet. Here I copy the NTLMv2 response and NTProofStr that will be used to build the crackable hash.
Server Challenge Value
From the NTLM server challenge packet, I copy the 8-byte challenge value. Combined with the response, this is what Hashcat needs for mode 5600 (NTLMv2).
Building hashes.txt
On the Windows machine I paste together the username, domain, challenge, and NTLMv2 response in the correct format (user::domain:challenge:response) and save them as hashes.txt.
Moving Hashes to Kali
Using a simple HTTP share, I browse to 192.168.1.2/hashes.txt from Kali and download the file. Now the attacker has all captured hashes offline with no risk of account lockouts.
Hashcat Attack Command
Running Hashcat in NTLMv2 mode (-m 5600) against hashes.txt with the rockyou.txt wordlist. This simulates a real attacker testing millions of common passwords.
Cracking Progress & Stats
Hashcat status view showing key metrics such as speed, progress through the keyspace, and candidates being tested. Weak passwords would quickly appear here as “Recovered”.
Authentication Context
Wireshark details of the NTLM Secure Service Provider including domain name, username, and workstation. This context ties each hash back to a real user account.

Key Takeaways

• NTLMv2 hashes can be captured by anyone who can see internal network traffic.
• Moving hashes offline allows attackers to brute-force without triggering lockouts.
• Long, unique passwords and multi-factor authentication dramatically reduce this risk.
